**Part A: Fix comparison link generation**

The DossierLayout generates comparison links at approximately line 623-637 using this pattern:
```astro
href={`/compare/${name.toLowerCase().replace(/\s+/g, '-')}-vs-${comp.toLowerCase().replace(/\s+/g, '-')}`}
```

This constructs URLs from the display `name` prop, which can mismatch actual comparison file slugs. For example:
- Peptide name "5-Amino-1MQ" -> slug would be "5-amino-1mq" (correct)
- But peptide name "GHK-Cu" -> slug would be "ghk-cu" (might not match comparison file if file uses "ghk" not "ghk-cu")

**Fix approach:** Instead of constructing comparison URLs from names, read the actual comparison collection to find matching files. The DossierLayout is already server-rendered, so it can use `getCollection`.

Add at the top of the frontmatter (after existing imports):
```typescript
import { getCollection } from 'astro:content';

// Get all comparisons to build a lookup for valid comparison links
const allComparisons = await getCollection('comparisons');
const comparisonSlugs = new Set(allComparisons.map(c => c.slug));
```

Then in the comparators section, change the link generation to:
```astro
{comparators.map(comp => {
  // Get the slug for this peptide (from the page's own slug)
  const mySlug = name.toLowerCase().replace(/\s+/g, '-');
  const compSlug = comp.toLowerCase().replace(/\s+/g, '-');

  // Try both directions to find the actual comparison file
  const forwardSlug = `${mySlug}-vs-${compSlug}`;
  const reverseSlug = `${compSlug}-vs-${mySlug}`;

  const actualSlug = comparisonSlugs.has(forwardSlug) ? forwardSlug
    : comparisonSlugs.has(reverseSlug) ? reverseSlug
    : null;

  if (!actualSlug) return null; // Skip if no comparison exists

  return (
    <a href={`/compare/${actualSlug}`} class="glass-card-hoverable p-6 flex items-center gap-4 group">
      {/* ... existing card content ... */}
    </a>
  );
})}
```

This ensures comparison links only appear when the comparison file actually exists, and uses the correct slug (handling both directions).

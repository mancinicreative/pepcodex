<task type="auto">
  <name>Task 1: Create Draft Generator n8n workflow</name>
  <files>n8n/workflows/draft-generator.json, n8n/docs/draft-generator.md</files>
  <action>
Create n8n workflow for MDX draft generation via Claude API.

**Workflow Structure:**

1. **Webhook Trigger** or **Manual Trigger**
   - Input: sourcePackPath (path to JSON)

2. **Read Source Pack** (Read File node)
   - Read JSON source pack

3. **Build Prompt** (Code node)
   - Construct Claude prompt with:
     - Peptide name and aliases
     - All sources with metadata
     - 12-section dossier template instructions
     - Safety constraints (NO dosing, NO sourcing)
     - Citation format instructions ([PMID:xxx])

4. **Claude API Call** (HTTP Request node)
   - URL: https://api.anthropic.com/v1/messages
   - Headers: x-api-key, anthropic-version
   - Body: model (claude-3-5-sonnet), max_tokens, messages
   - System prompt: "You are a scientific writer creating evidence-based peptide dossiers..."

5. **Extract MDX** (Code node)
   - Parse Claude response
   - Extract MDX content
   - Add frontmatter from source pack

6. **Output** (Set node or Write File)
   - Return MDX content for next workflow

**Prompt Template (embed in Code node):**
```
Create a comprehensive peptide dossier for {{peptide.name}}.

SOURCES AVAILABLE:
{{sources as formatted list}}

REQUIRED SECTIONS:
1. Quick Summary (5 bullets)
2. Identity & Origins
3. Mechanism of Action
4. Clinical Evidence
5. Safety Profile
6. Regulatory Status
7. Ongoing Research
8. Comparisons
9. Evidence Library (citation table)
10. Key Unknowns

CRITICAL RULES:
- NEVER include dosing, protocols, or administration instructions
- NEVER mention vendors, suppliers, or where to purchase
- ALWAYS cite sources as [PMID:xxxxx] or [NCT:xxxxx]
- CLEARLY label animal/in-vitro studies
- Use evidence-based language, avoid overclaims
```

**Documentation:** API setup, prompt customization, error handling
  </action>
  <verify>Workflow JSON is valid and importable</verify>
  <done>Draft generator workflow produces MDX matching site template</done>

<task type="auto">
  <name>Task 2: Create QA Gate n8n workflow</name>
  <files>n8n/workflows/qa-gate.json, n8n/docs/qa-gate.md</files>
  <action>
Create n8n workflow that runs QA validation.

**Workflow Structure:**

1. **Webhook Trigger**
   - Input: mdxPath, sourcePackPath

2. **Execute Command** (Execute Command node)
   - Command: `node /path/to/scripts/qa-validate.js {{mdxPath}} {{sourcePackPath}}`
   - Capture stdout as JSON

3. **Parse Results** (Code node)
   - Parse JSON output from qa-validate.js
   - Extract passed/failed status

4. **Branch: Pass/Fail** (IF node)
   - If passed: continue to output success
   - If failed: output failure with details

5. **Output Success** (Set node)
   - Return: {status: "passed", report: {...}}

6. **Output Failure** (Set node)
   - Return: {status: "failed", errors: [...], report: {...}}

**Error Handling:**
- If qa-validate.js crashes, catch and return error
- Include full error output for debugging

**Documentation:**
- How QA scripts are invoked
- Pass/fail criteria
- How to review failures
  </action>
  <verify>Workflow JSON is valid and references qa-validate.js correctly</verify>
  <done>QA gate workflow integrates validation scripts into pipeline</done>
</task>

**Workflow Structure:**

3. **Git Add** (Execute Command node)

4. **Git Commit** (Execute Command node)

5. **Git Push** (Execute Command node)
   - `git push origin main`

6. **Trigger Vercel Deploy** (HTTP Request node)
   - POST to Vercel deploy hook URL (from environment variable)

7. **Output** (Set node)
   - Return: {status: "published", peptide: slug, commitHash: ...}

**Error Handling:**
- If git commands fail, capture error and stop
- If Vercel hook fails, log but don't block (deploy may be triggered by push anyway)

**Documentation:**

- Vercel deploy hook creation
- Manual recovery steps

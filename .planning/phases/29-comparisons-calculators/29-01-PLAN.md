  <name>Task 2: Create comparison generation script and FAQ backfill script</name>
  <files>scripts/generate-comparisons.mjs, scripts/backfill-comparison-faqs.mjs</files>
  <action>
**Important:** The project does NOT have `gray-matter` installed. These scripts must parse YAML frontmatter manually using simple string splitting on `---` delimiters, or install gray-matter as a devDependency first. Recommend: `npm install --save-dev gray-matter` then use it.

**Script 1: `scripts/generate-comparisons.mjs`**

Create a Node.js ESM script that:

1. Reads all 92 peptide MDX files from `src/content/peptides/` and extracts frontmatter fields: `name`, `category`, `evidenceStrength`, `comparators`, `compoundType`
2. Reads all existing comparison files from `src/content/comparisons/` to build a set of existing pairs (normalized to alphabetical slug order to catch both directions)
3. Generates valid pairs using these rules:
   - Pair peptides that share the same `category` field
   - Pair peptides where one lists the other in its `comparators` array
   - EXCLUDE peptides with `compoundType` of 'mrna' or slugs containing oncology compounds (225ac-dota-lm3, bt5528, evx-01, lu-177, 177lu)
   - EXCLUDE pairs that already exist in either direction
   - Use alphabetical slug ordering for new pairs: if slug-A < slug-B alphabetically, file is `slug-A-vs-slug-B.mdx`
   - Cap at 180 new files maximum
4. For each pair, generates an MDX file with:
   - Frontmatter: title, peptideA, peptideB, category (use shared category, or peptideA's category for cross-category), lastUpdated (today's date), summary, metaTitle, metaDescription
   - 4 FAQs per comparison using templates filled with real peptide data:
     - Q1: "What is the main difference between {A} and {B}?" -> Answer based on their categories and compound types
     - Q2: "Which has more research evidence, {A} or {B}?" -> Answer using evidenceStrength fields
     - Q3: "Are {A} and {B} FDA approved?" -> Standard answer (neither is FDA approved for these uses)
     - Q4: "Can {A} and {B} be combined?" -> Standard cautionary research answer
   - Body sections: Overview, Evidence Comparison (table), Mechanism Comparison (table), Regulatory Status (table), Key Differences (table), Summary
   - Content should be substantive but templatized -- use actual peptide names and categories to fill templates. Each file should have at least 300 words of unique-ish content.

5. Writes files to `src/content/comparisons/`
6. Outputs a count of generated files and a summary

**Script 2: `scripts/backfill-comparison-faqs.mjs`**

Create a Node.js ESM script that:

1. Reads all existing comparison MDX files from `src/content/comparisons/`
2. Identifies files that do NOT have a `faqs:` field in their frontmatter (should be ~80 files based on research)
3. Reads the peptideA and peptideB slugs from each file's frontmatter
4. Looks up peptide names from `src/content/peptides/` frontmatter
5. Generates 4 templated FAQs (same template as generation script) and inserts them into the frontmatter YAML (before the closing `---`)
6. Writes the updated file back
7. Outputs count of files updated

Both scripts should:
- Use `import matter from 'gray-matter'` for reliable frontmatter parsing
- Have a `--dry-run` flag that prints what would happen without writing files
- Log progress to console
- Handle edge cases (missing peptide data, malformed frontmatter)

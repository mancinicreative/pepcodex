<task type="auto">
  <name>Task 2: Create Source Pack Builder n8n workflow</name>
  <files>n8n/workflows/source-pack-builder.json, n8n/docs/source-pack-builder.md</files>
  <action>
Create n8n workflow JSON for Source Pack Builder.

**Workflow Structure:**

1. **Manual Trigger** - Start with peptide name input
   - Parameters: peptideName, aliases (array)

2. **PubMed API Node** (HTTP Request)
   - URL: `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi`
   - Params: db=pubmed, term={peptideName}, retmax=50, retmode=json
   - Then fetch details: efetch.fcgi with returned IDs

3. **Europe PMC API Node** (HTTP Request)
   - URL: `https://www.ebi.ac.uk/europepmc/webservices/rest/search`
   - Params: query={peptideName}, resultType=core, format=json
   - Extracts: isOpenAccess flag

4. **ClinicalTrials.gov API Node** (HTTP Request)
   - URL: `https://clinicaltrials.gov/api/v2/studies`
   - Params: query.term={peptideName}, pageSize=20

5. **Merge Results Node** (Code node)
   - Combines PubMed + Europe PMC + ClinicalTrials.gov
   - Deduplicates by PMID/DOI
   - Formats to source pack schema

6. **Output Node** (Write to file or return JSON)

**Documentation (n8n/docs/source-pack-builder.md):**
- Workflow overview
- API endpoints used (with rate limits)
- Input parameters
- Output format
- Error handling notes

**Note:** n8n workflows are JSON exports. Create a working workflow structure that can be imported. Include all node configurations, connections, and parameters.
  </action>
  <verify>Workflow JSON is valid and can be imported into n8n (manual test)</verify>
  <done>Source Pack Builder workflow documented and exportable</done>
